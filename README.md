# Spider Solitaire

ブラウザで動作する Spider Solitaire（スパイダーソリティア）の静的 Web アプリです。  
Vite + TypeScript（フレームワークなし）で構築されており、任意の静的ホスティングサービスにデプロイできます。

## 環境構築手順

### 1. リポジトリをクローン

```sh
git clone https://github.com/denebola213/st-solitaire.git
cd st-solitaire
```

### 2. 依存パッケージをインストール

```sh
npm install
```

### 3. 開発サーバーを起動

```sh
npm run dev
```

ターミナルに表示された URL（例: `http://localhost:5173`）をブラウザで開くとアプリが表示されます。  
ファイルを編集すると自動的にブラウザが更新されます（Hot Module Replacement）。

---

## ビルド手順

### プロダクションビルド

```sh
npm run build
```

`dist/` ディレクトリに静的ファイルが出力されます。  
このフォルダをそのまま静的ホスティングサービス（GitHub Pages、Netlify、Vercel など）にデプロイできます。

### ビルド結果の確認

```sh
npm run preview
```

ビルド済みの `dist/` をローカルでプレビューするサーバーが起動します。

## プロジェクト構成

```
st-solitaire/
├── index.html                    # エントリーポイント HTML
├── vite.config.ts                # Vite 設定
├── tsconfig.json                 # TypeScript 設定
├── package.json
├── public/
│   └── patterns/
│       ├── easy.json             # イージー用プリセットパターン (10種)
│       ├── medium.json           # ミディアム用プリセットパターン (10種)
│       └── hard.json             # ハード用プリセットパターン (10種)
├── scripts/
│   └── generate-patterns.ts     # パターン生成スクリプト
└── src/
    ├── types.ts                  # 型定義 (Card, GameState, Pattern, Difficulty)
    ├── game.ts                   # ゲームロジック (移動検証・配牌・自動完成・勝利判定)
    ├── ui.ts                     # DOM レンダリング・ドラッグ＆ドロップ
    ├── main.ts                   # ゲーム/UI の連携・難易度選択・アンドゥ
    ├── patterns.ts               # パターンファイルの読み込み
    └── style.css                 # スタイル (モバイル対応レスポンシブ)
```

### 主要モジュールの概要

| ファイル | 役割 |
|---|---|
| `src/types.ts` | `Card`・`GameState`・`Pattern`・`Difficulty` の型定義 |
| `src/game.ts` | 移動検証・山札配布・K→A シーケンス自動完成・勝利判定 |
| `src/ui.ts` | DOM 描画・マウス/タッチ対応ドラッグ＆ドロップ・ゴーストカード・移動先ハイライト |
| `src/main.ts` | ゲーム/UI の連携・難易度選択・無制限アンドゥ |
| `src/patterns.ts` | `fetch` によるパターン JSON の非同期読み込み |
| `src/style.css` | CSS カスタムプロパティ・`@media (max-width: 768px)` によるモバイル対応 |

## パターンファイルの再生成

`public/patterns/` 以下の JSON は事前生成済みですが、以下のコマンドで再生成できます。

```sh
npm run generate-patterns
```

Fisher-Yates シャッフルで各難易度 10 種類のパターンを生成し、`public/patterns/{easy,medium,hard}.json` を上書きします。

## 利用者向けマニュアル

### ゲームの概要

Spider Solitaire は 2 デッキ（104 枚）のトランプを使うソリティアです。  
タブロー（場）の 10 列にカードを並べ、同じスートの K（キング）→ A（エース）の連続した 13 枚を揃えることを繰り返して、8 組すべて完成させたらクリアです。

---

### 難易度

| 難易度 | 使用スート | 特徴 |
|---|---|---|
| **Easy（イージー）** | スペード 1 種類 | スート統一でシーケンスが作りやすい |
| **Medium（ミディアム）** | スペード・ハート 2 種類 | 2 スートのため中程度の難易度 |
| **Hard（ハード）** | 4 スート（標準 2 デッキ） | 全スート使用で最も難しい |

---

### 画面説明

```
┌────────────────────────────────────────────────────────────┐
│  Moves: 0    Stock: 5    Completed: 0/8   [Undo] [New Game]│  ← ヘッダー
├──────────────────────────────────────┬─────────────────────┤
│                                      │  [山札]             │
│   タブロー（10 列）                  │                     │
│                                      │  [完成エリア]       │
│                                      │  (8 スロット)       │
└──────────────────────────────────────┴─────────────────────┘
```

| 要素 | 説明 |
|---|---|
| **タブロー** | カードを並べてシーケンスを作るメインエリア（10 列） |
| **山札 (Stock)** | クリックすると各列に 1 枚ずつ配布される（最大 5 回） |
| **完成エリア** | K→A のシーケンスが自動的に移動される場所（8 スロット） |
| **Moves** | 現在の手数 |
| **Stock** | 残り配布回数 |
| **Completed** | 完成済みシーケンス数（8 で勝利） |

---

### 操作方法

#### PC（マウス操作）

| 操作 | 内容 |
|---|---|
| カードをドラッグ＆ドロップ | 他の列へカードまたはシーケンスを移動 |
| 山札をクリック | 全列に 1 枚ずつカードを配布 |
| **Undo** ボタンをクリック | 1 手前の状態に戻す（無制限） |
| **New Game** ボタンをクリック | 難易度選択画面に戻る |

#### スマートフォン・タブレット（タッチ操作）

| 操作 | 内容 |
|---|---|
| カードを長押し＆スライド | 他の列へカードまたはシーケンスを移動 |
| 山札をタップ | 全列に 1 枚ずつカードを配布 |

---

### ゲームのルール

1. **カードの移動**  
   - 表向きカードを、1 ランク上のカードが一番上にある列にドラッグできます。  
   - 空の列にはどのカードでも移動できます。  
   - スートに関係なく数字が連続していれば移動できますが、**同一スートの連続序列のみ**まとめて複数枚移動できます。

2. **山札の配布**  
   - タブロー左側の山札をクリック/タップすると、各列の先頭に 1 枚ずつカードが配布されます。  
   - 空の列がある状態では山札を配布できません。  
   - 山札は最大 5 回配布できます。

3. **シーケンスの完成**  
   - 同一スートの K（13）→ Q（12）→ … → A（1）の 13 枚が揃うと自動的に完成エリアへ移動します。

4. **勝利条件**  
   - 8 組のシーケンスをすべて完成させるとクリアです。

---

### ヒント

- まず表向きになっていないカードを早めに開けることを優先しましょう。
- 同じスートでシーケンスを作ると、まとめて移動できるので有利です。
- アンドゥは無制限なので、試行錯誤しながら進めましょう。
- 空の列は移動先として非常に有効なので大切にしましょう。
- 山札の配布は最終手段として残しておくと攻略しやすくなります。
